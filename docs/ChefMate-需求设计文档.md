## 1. 项目概述

### 1.1 项目背景

随着生活节奏加快，越来越多的年轻人和家庭面临"不知道吃什么"、"食材浪费严重"、"想做饭却不会"等痛点。市场上虽然有一些食谱类应用，但大多停留在静态内容展示阶段，缺乏智能化、个性化和协作化能力。

**ChefMate（厨友）** 定位为一款 **AI 驱动的智能家庭烹饪助手**，通过拍照识别食材、AI 智能推荐食谱、沉浸式烹饪指引、视频/音频智能解析等核心能力，帮助用户从"买菜→备菜→做菜→饮食管理"实现全流程智能化。同时支持家庭协作，让家庭成员之间可以高效分工配合。

### 1.2 项目目标

- **降低做饭门槛**：让烹饪小白也能轻松上手，告别外卖依赖
- **减少食材浪费**：通过智能入库和过期预警，最大化食材利用率
- **提升家庭协作效率**：支持多人分工，买菜、备菜任务一键分配
- **打造沉浸式烹饪体验**：AI 对话式指引 + 智能倒计时，让做菜过程更轻松
- **连接灵感与实践**：看到美食视频即可一键解析生成食谱，从"想吃"到"做出来"零障碍

### 1.3 目标用户

| 用户画像           | 描述                                                 |
| ------------------ | ---------------------------------------------------- |
| **年轻家庭**       | 20-35 岁，有一定健康饮食意识，希望减少外卖、在家做饭 |
| **上班族**         | 工作日时间紧张，需要快速决定吃什么、高效完成烹饪     |
| **全职父母**       | 负责家庭饮食，需要菜谱灵感和营养搭配建议             |
| **烹饪小白**       | 完全不会做饭，需要手把手指引和详细步骤               |
| **外卖转自炊人群** | 想要改变饮食习惯，从点外卖转为自己做饭的人群         |
| **独居青年**       | 一个人生活，食材容易浪费，需要个性化小份食谱推荐     |

### 1.4 核心价值

- **智能**：AI 贯穿全流程——识别食材、推荐食谱、解析视频、语音指引
- **省心**：不用再纠结"今天吃什么"，根据现有食材一键生成方案
- **协作**：家庭成员分工明确，买菜、做饭不再是一个人的事
- **有趣**：沉浸式烹饪体验让做饭变成一种享受

### 2.2 用户故事

#### 故事 1：视频识别复刻美食

- **作为** 一个喜欢刷美食视频的用户
- **我想要** 将抖音看到的做饭视频导入 ChefMate，自动识别并生成食谱
- **以便** 我可以按照结构化的步骤轻松复刻这道菜
- **验收标准**：
  - [ ] 支持导入本地视频文件
  - [ ] AI 能识别视频中的食材和烹饪步骤
  - [ ] 生成的食谱包含完整的食材清单和分步操作
  - [ ] 自动关联冰箱库存，标记已有和缺少的食材

#### 故事 2：食材驱动的智能做饭

- **作为** 一个不知道今天吃什么的用户
- **我想要** 查看冰箱里现有的食材，让 AI 帮我推荐能做的菜
- **以便** 我可以快速决定做什么，不浪费已有食材
- **验收标准**：
  - [ ] 可以查看冰箱所有食材列表
  - [ ] 支持手动勾选想用的食材
  - [ ] AI 在 5 秒内生成至少 3 个食谱推荐
  - [ ] 每个推荐包含难度标签、预计烹饪时间和营养信息

#### 故事 3：沉浸式烹饪体验

- **作为** 一个烹饪小白
- **我想要** 在做菜过程中获得语音指引和智能倒计时提醒
- **以便** 我不用反复看手机，双手可以专注于烹饪操作
- **验收标准**：
  - [ ] 提供分步语音播报烹饪指令
  - [ ] 支持语音对话，可以问"下一步是什么"
  - [ ] 自动设置焖煮、腌制等环节的倒计时
  - [ ] 倒计时结束时发出语音和震动提醒

#### 故事 4：家庭协作买菜

- **作为** 一个家庭组的管理员
- **我想要** 根据本周食谱生成购物清单，并分配给家人去采购
- **以便** 家庭采购任务分工明确，不会遗漏或重复购买
- **验收标准**：
  - [ ] 一键根据食谱生成购物清单
  - [ ] 支持将清单中的商品分配给不同家庭成员
  - [ ] 被分配人收到任务通知
  - [ ] 采购完成后可标记已买，其他成员实时可见

#### 故事 5：食材智能管理

- **作为** 一个注重食材新鲜度的用户
- **我想要** 拍照录入食材后系统自动跟踪保质期并提醒我
- **以便** 我能及时消耗即将过期的食材，减少浪费
- **验收标准**：
  - [ ] 拍照后 AI 识别食材种类和大致数量
  - [ ] 自动推算各类食材的建议保质期
  - [ ] 过期前 1-3 天推送预警通知
  - [ ] 优先推荐使用即将过期食材的食谱

### 2.3 功能详细说明

#### 功能 1：拍照智能入库

**描述**：用户拍摄食材照片，AI 自动识别食材种类、数量（**目标准确率 ≥85%**），录入冰箱库存系统，并根据食材类型自动推算保质期。**支持双模式识别**：若照片中包含商品标签（如包装袋、瓶身标签），优先通过 OCR 读取标签信息（名称、生产日期、保质期等）；若无标签，则直接通过 AI 视觉识别食材。

**准确率衡量方式**：通过用户修正率统计——若用户对识别结果未做修改直接确认入库，则视为识别正确。每周统计一次准确率，低于目标值时触发模型优化。

**输入**：用户拍摄的食材照片（支持单张/多张）

**处理**：

1. AI 视觉模型预判照片中是否包含商品标签
2. **有标签路径**：OCR 识别标签文字 → 提取食材名称、生产日期、保质期、规格等精确信息
3. **无标签路径**：AI 视觉模型直接识别食材种类和估算数量 → 匹配食材数据库推算默认保质期
4. 创建库存记录，关联到用户所属家庭组
5. 根据食材类型设置过期预警时间

**输出**：识别结果列表（食材名称、数量、保质期、识别来源标记[标签/图像]），用户确认后自动入库

**异常处理**：

- 标签模糊/破损：降级为图像识别模式
- 识别失败：提供手动输入入口
- 识别不准确：支持用户修改识别结果
- 网络异常：缓存照片，恢复网络后重试

#### 功能 2：AI 智能生成食谱

**描述**：基于用户冰箱现有食材、个人偏好（口味、过敏源、饮食禁忌）和家庭成员信息，通过 AI 流式生成个性化食谱推荐。

**输入**：用户选择的食材列表 + 个人/家庭偏好设置

**处理**：

1. 汇总所选食材和用户偏好信息
2. 构建 Prompt 调用 AI 大模型（SSE 流式输出）
3. 解析返回内容为结构化食谱数据
4. 补充营养信息和难度标签

**输出**：3-5 个食谱推荐（含食材清单、步骤、营养信息、预计烹饪时间）

**异常处理**：

- AI 响应超时：展示缓存的热门食谱
- 食材过少：提示用户可搭配采购的食材建议
- 内容不当：输出内容安全过滤

#### 功能 3：视频/音频识别生成食谱

**描述**：用户导入美食视频或音频文件，AI 自动分析视频画面和语音内容，提取食材信息和烹饪步骤，生成结构化食谱。

**输入**：用户上传的视频文件或音频文件

**输入约束**：

| 约束项       | 限制条件                                                         |
| ------------ | ---------------------------------------------------------------- |
| 视频格式     | MP4、MOV                                                         |
| 音频格式     | MP3、WAV、M4A                                                    |
| 最大文件大小 | 500MB                                                            |
| 最大时长     | 30 分钟                                                          |
| 在线链接     | 暂不支持（v1.0 仅支持本地文件导入，后续版本可扩展支持 URL 解析） |

**处理**：

1. 文件校验：检查格式、大小和时长是否符合约束
2. 视频：提取关键帧 + 音轨分离
3. 音频：语音转文字（ASR）
4. AI 多模态分析：结合画面和文字内容识别食材和步骤
5. 生成结构化食谱数据

**输出**：完整食谱（食材清单 + 分步操作 + 预计时间 + 注意事项）

**异常处理**：

- 文件超出限制：弹窗提示具体限制条件，建议用户压缩或截取片段
- 视频过长（>30分钟）：提示用户截取关键片段
- 识别准确度低：允许用户手动编辑修正
- 不支持的格式：明确提示支持的格式列表（MP4/MOV/MP3/WAV/M4A）

#### 功能 4：沉浸式烹饪指引

**描述**：进入烹饪模式后，系统以分步卡片 + 语音播报的方式引导用户完成烹饪，支持语音对话交互和智能倒计时。

**输入**：用户选择的食谱

**处理**：

1. 将食谱拆解为分步烹饪指令
2. 依次展示每步操作卡片并语音播报
3. 监听用户语音指令并执行对应操作
4. 自动识别需要计时的步骤并设置倒计时

**支持的语音指令词表**：

| 指令类别  | 语音关键词                     | 功能说明                            |
| --------- | ------------------------------ | ----------------------------------- |
| 步骤导航  | "下一步"、"继续"               | 跳转到下一烹饪步骤                  |
| 步骤导航  | "上一步"、"返回"               | 返回上一烹饪步骤                    |
| 重复      | "重复"、"再说一遍"             | 重新播报当前步骤                    |
| 暂停/恢复 | "暂停"、"继续播放"             | 暂停/恢复语音播报                   |
| 倒计时    | "设置倒计时X分钟"、"计时X分钟" | 手动设置计时器                      |
| 查询      | "这步要多久"、"还有多久"       | 查询当前步骤时长或剩余倒计时        |
| 查询      | "需要什么材料"、"用量多少"     | 查询当前步骤所需食材和用量          |
| 自由对话  | 任意烹饪相关问题               | AI 理解并回答（如"火候不够怎么办"） |

> **说明**：v1.0 优先实现指令词匹配模式（前 7 类），自由对话模式（第 8 类）作为增强能力在后续版本迭代。

**输出**：分步操作卡片 + 语音播报 + 倒计时提醒 + 完成提示

**异常处理**：

- 语音识别失败：提供手动按钮操作（UI 上同步展示所有指令按钮）
- 环境噪音过大：自动提升灵敏度阈值，或弹出提示建议切换手动模式
- 倒计时超时未操作：自动暂停并语音提醒
- 手机息屏：支持后台语音播报和倒计时

## 3. 非功能需求

### 3.1 性能要求

| 指标         | 目标值                   | 说明                     |
| ------------ | ------------------------ | ------------------------ |
| 页面加载时间 | < 2 秒                   | 主要页面首屏加载         |
| AI 食谱生成  | < 5 秒                   | 首个内容输出时间（流式） |
| 拍照识别     | < 3 秒                   | 食材识别结果返回         |
| 并发用户数   | 初期 1,000，后期 100,000 | 需支持弹性扩容           |
| API 响应时间 | < 500ms                  | 非 AI 类接口             |
| 视频解析     | < 30 秒                  | 10 分钟以内的视频        |

### 3.2 安全要求

- **身份认证**：JWT Token 认证，支持 Token 自动刷新
- **数据加密**：敏感数据传输使用 HTTPS，密码使用 bcrypt 加密存储
- **权限控制**：基于家庭组的数据隔离，强制校验 `group_id`
- **隐私保护**：用户照片、视频等仅用于识别目的，不做其他用途存储
- **输入校验**：所有 API 使用 `class-validator` 严格校验 DTO

### 3.3 兼容性要求

| 平台    | 最低版本              |
| ------- | --------------------- |
| iOS     | iOS 13+               |
| Android | Android 8.0+ (API 26) |

### 3.4 可用性要求

- **系统可用性**：99.5%（初期目标）
- **容错能力**：AI 服务不可用时降级为缓存推荐
- **备份恢复**：数据库每日自动备份，支持 7 天内恢复
- **离线支持**：已缓存的食谱和库存数据在无网络时仍可查看（规划中）

### 3.5 数据迁移与版本升级

- **数据库迁移**：使用 TypeORM/Prisma 内置的 Migration 机制，所有表结构变更必须通过迁移脚本执行，禁止手动修改线上数据库
- **版本兼容**：API 接口遵循向后兼容原则，破坏性变更需要发布新版本号（如 `/api/v2/`）
- **客户端升级**：通过版本号比对提示用户更新，重大变更时强制更新
- **数据回滚**：每次迁移需提供 rollback 脚本，确保升级失败时可安全回退

## 4. 技术方案

### 4.1 技术栈

| 层级          | 技术选型                          | 说明                                   |
| ------------- | --------------------------------- | -------------------------------------- |
| **前端框架**  | Uni-app (Vue 3 + Composition API) | 一套代码发布 iOS、Android              |
| **前端语言**  | TypeScript（严格模式）            | 类型安全，减少运行时错误               |
| **状态管理**  | Pinia                             | Vue 3 官方推荐                         |
| **UI 组件库** | uni-ui / uView Plus               | Uni-app 生态成熟组件                   |
| **CSS 方案**  | TailwindCSS + Sass/SCSS           | 原子化 CSS + 预处理器                  |
| **后端框架**  | NestJS (Node.js)                  | 模块化架构，TypeScript 全栈            |
| **ORM**       | TypeORM / Prisma                  | 类型安全的数据库操作                   |
| **数据库**    | PostgreSQL                        | 支持 JSON 字段，适合存储食谱结构化数据 |
| **缓存**      | Redis                             | 分布式锁、会话缓存、任务队列           |
| **AI 大模型** | DeepSeek / OpenAI                 | 食谱生成、食材识别、多模态理解         |
| **语音服务**  | TTS/ASR 引擎                      | 语音播报和语音识别                     |
| **国际化**    | i18n 预留（vue-i18n）             | 当前仅中文，架构预留多语言扩展能力     |

### 4.2 系统架构

**整体架构采用前后端分离的 C/S 架构：**

| 层级           | 组成               | 职责                                               |
| -------------- | ------------------ | -------------------------------------------------- |
| **客户端层**   | Uni-app 移动端     | 用户交互、拍照、语音、视频导入                     |
| **API 网关层** | NestJS Gateway     | 路由分发、JWT 鉴权、限流、日志                     |
| **业务服务层** | NestJS Modules     | 用户模块、食材模块、食谱模块、家庭组模块、任务模块 |
| **AI 服务层**  | AI Service         | 食材识别、食谱生成、视频/音频解析（SSE 流式）      |
| **数据存储层** | PostgreSQL + Redis | 持久化存储 + 缓存 + 消息队列                       |
| **定时任务层** | CronService        | 食材过期扫描、数据统计报告生成                     |

### 4.3 数据模型

**核心实体关系：**

| 实体                           | 关键字段                                                                                                    | 关系              |
| ------------------------------ | ----------------------------------------------------------------------------------------------------------- | ----------------- |
