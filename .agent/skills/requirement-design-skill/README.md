## 🎯 核心功能

### 1. 交互式问答收集

通过结构化的问题引导用户提供完整的需求信息：

- 项目基础信息(名称、类型、背景、用户)
- 核心功能需求(功能列表、使用场景)
- 技术和约束(技术栈、性能、兼容性)
- 项目规划(时间、团队、优先级)

### 2. 智能优化补充 🧠

基于项目类型和已有信息,智能推断可能缺失的功能：

- 自动识别常见功能模式
- 补充非功能需求
- 提醒潜在的技术风险

### 3. 提出优化建议 💡

从多个维度提供专业建议：

- SMART原则检查(具体、可衡量、可实现、相关、有时限)
- 识别模糊或不明确的描述
- 按优先级(高/中/低)提供改进建议

### 4. AI智能评估 📊

多维度评估需求文档质量：

- **完整性评分**(0-100)：检查必需字段是否完整
- **清晰度评分**(0-100)：评估描述的明确性
- **可行性评分**(0-100)：评估技术和资源可行性
- **综合评分**：生成详细的评估报告

### 5. PDF 自动导出 📕

文档生成后自动询问是否需要导出 PDF：

- **智能路径处理**：自动识别用户提供的路径或使用默认路径
- **自动调用转换**：集成 `markdown-to-pdf-skill` 进行转换
- **专业样式**：自动应用目录、代码高亮、A4纸张等专业设置
- **灵活选择**：用户可以选择立即导出或稍后导出

## 📖 使用流程

```
1. 启动 Skill
   ↓
2. 回答10-15个结构化问题
   ↓
3. AI智能补充可能缺失的功能
   ↓
4. 获得优化建议(高/中/低优先级)
   ↓
5. 查看质量评估报告
   ↓
6. 获得完整的需求设计文档（Markdown）
   ↓
7. 📕 PDF 导出询问（自动）
   ├─ 是 → 自动转换为 PDF
   └─ 否 → 保持 Markdown 格式
   ↓
8. 迭代优化(可选)
```

## 📄 生成的文档包含

1. **项目概述**
   - 项目背景、目标、用户画像、核心价值

2. **功能需求**
   - 功能列表、用户故事、详细说明、验收标准

3. **非功能需求**
   - 性能、安全、兼容性、可用性要求

4. **技术方案**
   - 技术栈、系统架构、数据模型、接口设计

5. **项目计划**
   - 开发阶段、里程碑、资源配置、风险评估

6. **测试和部署**
   - 测试策略、部署方案、监控方案

7. **附录**
   - 术语表、参考资料、变更记录

### 优化建议示例

```
🔴 高优先级
- 功能描述过于模糊 → 添加具体的验收标准

🟡 中优先级
- 缺少性能指标 → 补充响应时间、并发数等

🟢 低优先级
- 考虑移动端适配 → 建议响应式设计
```

### 评估报告示例

```
📊 总体评分：78/100 ⭐⭐⭐⭐

完整性：32/40 (良好)
清晰度：24/30 (良好)
可行性：22/30 (及格)

主要优点：
1. 核心功能定义清晰
2. 技术栈选择合理

需要改进：
1. 补充验收标准
2. 细化性能指标
```

## 🎓 适用场景

✅ **适合使用**

- 启动新项目前需要整理需求
- 将零散的想法转化为结构化文档
- 需要对现有需求进行完善和优化
- 需要评估需求的完整性和可行性

❌ **不适合使用**

- 需求已经非常明确且文档完善
- 只是简单的功能修改
- 紧急的bug修复

## 💬 常见问题

**Q: 生成文档需要多长时间？**  
A: 通常10-20分钟,取决于项目复杂度和回答详细程度。

**Q: 可以中途暂停吗？**  
A: 可以,AI会记住已收集的信息,下次继续。

**Q: 生成的文档可以导出吗？**  
A: 可以！文档生成后会自动询问是否需要导出为 PDF。你可以选择立即导出或稍后导出。文档是标准的 Markdown 格式，也可以手动转换为 Word 等其他格式。

**Q: 可以为现有项目生成文档吗？**  
A: 可以,只需提供现有项目的信息即可。
